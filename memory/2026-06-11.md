# Memory Log - 2026-06-11

## Objective
Fix the issue where clicking the "GET JD FROM PAGE" button on LinkedIn jobs page extracts the wrong job description (specifically matching the first job card in the left list area instead of the selected job in the right details area).

## Root Cause
- The generic extraction code in the message handlers (`content.js`, `popup.js`, `background.js`) looped over selectors (like `.show-more-less-html__markup`, `.jobs-description__content`) globally on `document` tree.
- Since LinkedIn search page DOM retains elements for list items at the beginning of the tree before the right details pane, `document.querySelector` matched the first job card in the list rather than the active selection.
- Furthermore, the generic extraction logic did not prioritize the portal-specific handlers (like `PortalHandlers.LinkedIn`).

## Solution
1. Scoped `getJobDescription()` and `getJobInfo()` inside the active details pane container in `content/portal-handlers/linkedin.js` (e.g. searching inside `.jobs-search-two-pane__details` or `.jobs-search__job-details--container` first) to prevent matching the left listing sidebar/cards.
2. Updated `CLYDE_EXTRACT_REQUEST` and `EXTRACT_JD_FROM_PAGE` message handlers in `content/content.js` to detect a registered portal handler (like LinkedIn) and prioritize its custom `getJobDescription()` function before falling back to generic query selectors.
3. Updated the scripting fallbacks in `popup.js` and `background.js` to also check for `PortalHandlers` and prioritize the matched handler's custom extraction method.
